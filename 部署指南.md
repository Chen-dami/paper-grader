## 一、快速开始（只需 3 步）

### 方式 A：双击安装（推荐 · 新手友好）

```
1. 解压 ai-grader.zip 到任意目录
2. 双击 一键安装.bat → 等待 3-5 分钟
3. 配置 API Key（见下文）
4. 双击 启动阅卷系统.bat → 浏览器自动打开
```

### 方式 B：命令行安装（适合开发者）

```bash
# 1. 克隆仓库
git clone <仓库地址> ai-grader
cd ai-grader

# 2. 创建环境 + 安装依赖
python -m venv .venv
.venv\Scripts\python -m pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 3. 启动
.venv\Scripts\streamlit run app.py
```

## 二、API Key 配置

### 获取 Key

| 平台 | 地址 | 模型 | 用途 | 费用 |
|------|------|------|------|------|
| **DeepSeek** | [platform.deepseek.com](https://platform.deepseek.com) | deepseek-chat | 文本评分 | ¥1/百万 tokens |
| **智谱** | [open.bigmodel.cn](https://open.bigmodel.cn) | glm-4v-flash | 免费视觉（1图） | **免费** |
| 阿里通义 | [dashscope.console.aliyun.com](https://dashscope.console.aliyun.com) | qwen-vl-plus | 付费视觉（5图） | ¥1.5/百万 tokens |
| OpenAI | [platform.openai.com](https://platform.openai.com) | gpt-4o-mini | 视觉备选 | $0.15/百万 tokens |

### 配置方式

**方式 A：环境变量（推荐）**
```batch
# 打开 cmd，执行：
setx DEEPSEEK_KEY "你的DeepSeek_API_Key"
setx ZHIPU_KEY "你的智谱API_Key"        ← 推荐（免费视觉）
setx BAILIAN_KEY "你的阿里云API_Key"     ← 可选（付费视觉，更准）

# 配置后需重新打开终端窗口生效
```

**方式 B：.env 文件**
```
1. 将 .env.example 改名为 .env
2. 编辑 .env，填入你的 Key
```

> **最低配置**：DeepSeek（文本）+ 智谱（免费视觉），即可覆盖全部功能且视觉免费。

## 三、视觉策略配置

系统支持三种视觉策略，可在 Web 界面或 AI 助手中切换：

| 策略 | 视觉模型 | 看图数 | 费用 | 评分风格 |
|------|---------|--------|------|---------|
| 🆓 免费视觉 | glm-4v-flash | 1张/题 | 免费 | 适中 |
| 💰 付费视觉 | qwen-vl-plus | 5张/题 | ~¥0.01/题 | 严格 |
| 📝 纯文字 | 无 | 0 | ~¥0.002/题 | 宽松 |

**切换方式：**
- Web 界面：「评分配置 → 模型路由」下拉框选择
- AI 助手：「阅卷控制」模式，说"换成免费视觉"即可
- 阅卷页：每题旁有复选框，可单独开关视觉

## 七、常见问题

### 安装问题

**Q: 双击 .bat 闪退？**
A: 在脚本所在目录的地址栏输入 `cmd` 回车，手动输入脚本名运行，就能看到错误信息。

**Q: pip 安装慢 / 超时？**
A: 脚本已配置清华镜像。如仍失败，手动执行：
```batch
.venv\Scripts\pip install -r requirements.txt -i https://mirrors.aliyun.com/pypi/simple/
```

**Q: 端口 8501 被占用？**
A: `启动阅卷系统.bat` 会自动释放端口。如果仍占用，指定其他端口：
```batch
.venv\Scripts\streamlit run app.py --server.port 8502
```

### 模型问题

**Q: 提示 "模型不可用"？**
A: 进入「评分配置 → 模型路由」→ 点击「🔍 检测API Key」手动检测。查看各模型状态和可用性。

**Q: 为什么有些模型检测不到？**
A: 检查环境变量：`echo %ZHIPU_KEY%`。配置后需**重新打开 cmd/终端**才生效。

**Q: 智谱视觉不工作？**
A: `glm-4v-flash` 免费版可能有频率限制，批量阅卷时稍等片刻。也可切换用阿里云 `qwen-vl-plus`。

**Q: deepseek-chat 不支持视觉？**
A: 对，deepseek-chat 是纯文本模型。视觉需要智谱或阿里云。系统会自动检查并 fallback：如果配置的 vision_model 不支持视觉，会自动切换到可用的视觉模型。

### 评分问题

**Q: 为什么同一份卷子不同策略分数差异很大？**
A: 这是正常的。纯文字模式宽松（看不到图不扣分），付费视觉严格（看5张图全面评估）。建议日常用免费视觉，正式评分用付费视觉。

**Q: 视觉题得分异常低？**
A: 可能是图片内容偏题或质量差。运行 `debug_grade.py` 查看视觉模型的图片描述和 LLM 的完整评分过程。

**Q: 空题却得了分？**
A: 纯文字模式下的设计行为——表格中有截图标记就视为已提交。切换到免费/付费视觉可避免。

**Q: AI 助手改了设置怎么恢复？**
A: 「阅卷控制」模式的修改是**会话临时**的，在阅卷页点"恢复默认"或刷新页面即复原 config.yaml 的默认值。

## 八、项目目录结构

```
ai-grader/
├── 一键安装.bat           # 全自动安装向导
├── setup.bat              # 虚拟环境安装
├── 启动阅卷系统.bat       # 日常启动
├── 更新.bat               # Git 自动更新
├── 给AI的安装提示词.txt    # 复制给 AI 帮你装
├── app.py                 # Web 入口
├── main.py                # CLI 入口
├── config.yaml            # 模型路由/评分配置
├── requirements.txt       # Python 依赖
├── .env.example           # API Key 模板
├── src/                   # 源代码
│   ├── grader.py          # 评分引擎（两阶段流水线）
│   ├── model_router.py    # 模型路由 + 策略管理
│   ├── preprocessor.py    # 表格匹配 + 字段提取
│   ├── grader_strategies.py  # 三种视觉策略
│   └── grading_overrides.py  # AI 助手会话覆盖
├── pages/                 # Streamlit 页面
│   ├── 1_grading.py       # 阅卷
│   ├── 2_config.py        # 评分配置
│   ├── 3_results.py       # 成绩查看
│   ├── 4_ai_assistant.py  # AI 助手
│   └── ...
├── data/                  # 试卷 + 评分标准 + 用户数据
├── output/                # 评分结果
├── tools/                 # 诊断/调试工具
│   ├── debug_grade.py     # 单卷完整评分链路
│   ├── scan_class.py      # 批量扫描班级
│   ├── batch_class_test.py   # 批量评分测试
│   └── diagnose_anomaly.py   # 异常分数诊断
└── tests/                 # 单元测试（270+ 用例）
```
