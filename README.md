# AI 智能阅卷系统

LLM 驱动的多题型、多模态作业自动批阅系统。支持文字/图像/视频/Excel/智能体五种题型，三种视觉策略可切换，AI 助手可自然语言调整评分设置。**两阶段评分流水线（视觉描述 → 文本评分）省 70-90% API 费用**。

---

## 快速开始

```
1. 双击 "一键安装.bat" → 等待3-5分钟
2. 配置 API Key（打开 cmd 执行）:
   setx DEEPSEEK_KEY "你的DeepSeek Key"      ← 必需（文本评分）
   setx ZHIPU_KEY "你的智谱Key"              ← 推荐（免费视觉）
   setx BAILIAN_KEY "你的阿里云Key"           ← 可选（付费视觉）
3. 双击 "启动阅卷系统.bat" → 浏览器自动打开 http://localhost:8501
4. 进入「评分配置 → 模型路由」→ 点击「🔍 检测API Key」
```

> 详细说明见 [`部署指南.md`](部署指南.md)

## 三种视觉策略

系统支持三种视觉策略，在「评分配置 → 模型路由」或 AI 助手中随时切换：

| 策略 | 视觉能力 | 费用 | 适用场景 |
|------|---------|------|---------|
| 🆓 **免费视觉** | 1张图/题（glm-4v-flash） | 零费用 | 日常批改、快速检查 |
| 💰 **付费视觉** | 5张图/题（qwen-vl-plus） | ~¥0.01/题 | 正式考试、需精细评估 |
| 📝 **纯文字** | 不看图，只看文字 | ~¥0.002/题 | 只看文字质量、最快 |

**策略如何影响评分：**

| 策略 | 评分风格 |
|------|---------|
| 纯文字 | 🌈 宽松——"看不到图不是学生的错"，有截图标记即视为已提交 |
| 免费视觉 | ⚖️ 适中——1张图辅助判断，平衡准确与成本 |
| 付费视觉 | 🔍 严格——5张图全面评估，对质量差最不手软 |

> 实测：同一份低质量卷子，三种策略下得分分别为 **51 / 35 / 27** 分。

**按题切换：** 阅卷页还支持逐题开关视觉——比如 Q2 让 AI 看图，Q3 老师自己看。

## 评分流水线（省钱核心）

```
学生作业 .docx
    │
    ├─→ 提取图片/视频帧
    │       │
    │       ▼
    │   视觉模型（qwen-vl-plus / glm-4v-flash）
    │   只描述不评分："看到了什么"
    │   输出 ~150 tokens ≈ ¥0.0007
    │       │
    │       ▼ 文字描述
    │       │
    └─→ 提取文字内容 ──→ DeepSeek-Chat
                         综合描述+文字 → 评分 JSON
                         输出 ~500 tokens ≈ ¥0.002

对比：让视觉模型直接评分 → 输出 ~2000 tokens ≈ ¥0.03/次
两阶段方案 → ~¥0.003/次，省 90%
```

## AI 助手（🤖 阅卷控制）

AI 助手支持**自然语言调整评分设置**，在「AI 助手」页选择「阅卷控制」模式：

```
你: "帮我把视觉策略换成免费的，评分放松一点"
AI: "好的，视觉策略→免费视觉，评分模式→宽松 ✅"
     → 设置即时生效，切到阅卷页即可看到效果

你: "恢复默认设置"
AI: "已恢复 config.yaml 的默认配置 ✅"
```

三种模式：

| 模式 | 做什么 | 生效 |
|------|--------|------|
| 💬 自由对话 | 聊天咨询 | — |
| 🎛️ 阅卷控制 | 调视觉策略、评分松严 | **即时生效** |
| ⚙️ 评分配置 | 改评分标准、档位、分值 | 预览→确认→永久写入 |

## 评分题型

| 类型 | 评分方式 | 说明 |
|------|---------|------|
| text | DeepSeek 文字评分 | 文本生成、文案创作 |
| vision | 视觉描述 + DeepSeek 评分 | 海报设计、视频创作（自动提取帧） |
| code | pandas 规则引擎 | Excel 数据处理（零 LLM 费用） |
| hybrid | DeepSeek + 元数据 | 智能体搭建（人设+知识库+链接） |
| multiple_choice / true_false / fill_blank / short_answer | 精确匹配 + 关键词 | 客观题 |

## 评分核心原则

1. **证据驱动** — 先检测学生提交了哪些材料（文字/图片/视频/Excel/链接），再据此评分
2. **技术限制不是学生错** — 纯文字模式下看不到图片不扣分；有截图标记即视为已提交
3. **空题零容忍** — 真正没交（无文字+无截图+无文件+无链接）才给0分
4. **按需松严** — 三种视觉策略覆盖从"宁可错给"到"力求精准"的完整谱系

## 常见问题

### 视觉策略相关

**Q: 三种视觉策略怎么选？**
A: 日常用「免费视觉」性价比最高；正式考试/需要精细评估用「付费视觉」；只想看文字质量用「纯文字」。切换在「评分配置 → 模型路由」下拉框或 AI 助手说一句即可。

**Q: 纯文字模式会不会太宽松？**
A: 是的，设计如此。"学生提交了材料但因系统技术限制看不到，不是学生的错"。实测纯文字模式平均92分，付费视觉平均79分。适合快速摸底，不适合精细排名。

**Q: 免费视觉（glm-4v-flash）够用吗？**
A: 实测够用。1张图就能发现大部分偏题和缺失问题。与付费视觉（5张图）的评分差异约6分（平均），精细度略低但零费用。

**Q: 能逐题开关视觉吗？**
A: 能。阅卷页每题旁有复选框，取消勾选则该题不走视觉API，老师自己看图打分。

### 评分相关

**Q: 启动后看不到模型？评分失败？**
A: 进入「评分配置 → 模型路由」→ 点击「🔍 检测API Key」按钮。系统需要手动检测（避免被安全软件误判）。

**Q: 视觉题得分很低或0分？**
A: 检查是否有视觉模型可用。至少配一个：智谱 `ZHIPU_KEY`（免费）或阿里云 `BAILIAN_KEY`（付费）。

**Q: 为什么有些题明明是空的却得了分？**
A: 纯文字模式下，表格中只要有截图标记就视为已提交——系统看不到图不能怪学生。切换到付费视觉即可严格评估。

**Q: API 费用大概多少？**
A: 一份试卷（5道题）约 ¥0.01-0.05。免费视觉方案零视觉费用。代码题零 LLM 费用。

**Q: 为什么视觉模型不直接评分？**
A: 视觉模型输出 token 很贵。两阶段流水线：视觉模型→描述（~150 tokens），DeepSeek→评分（~500 tokens），省 70-90%。

### AI 助手相关

**Q: AI 助手能做什么？**
A: 三种模式：①自由对话聊天 ②**阅卷控制**——说句话就能切换视觉策略/评分松严 ③评分配置——修改评分标准/档位/分值。

**Q: AI 助手改了设置，会影响 config.yaml 吗？**
A: 「阅卷控制」的修改是**会话临时**的，不影响 config.yaml，刷新或点"恢复默认"即复原。「评分配置」模式才会永久写入文件。

**Q: 怎么调试某份卷子的评分？**
A: 命令行运行 `python tools/debug_grade.py "学生.docx路径" --strategy free_vision`，会显示每次 LLM 请求的完整内容和返回结果。

### 其他

**Q: 查重怎么用？**
A: 阅卷时勾选「阅卷后自动查重」。可疑度 ≥300 自动判零。报告在 output/班级名/ 下。

**Q: 如何批量评分？**
A: 命令行：`python main.py --workers 8 --check`

**Q: 换电脑怎么迁移？**
A: 复制整个文件夹到新电脑，双击「一键安装.bat」。`data/` 和 `output/` 保留，`.venv/` 不需要复制。
